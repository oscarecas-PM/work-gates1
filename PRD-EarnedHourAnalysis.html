Product Requirements Document (PRD)

Product Name: Work-Order Hours Earned Bundle Viewer (Earned Hour Analysis)

File: earned-hour-analysis-1.html

Version 1.2 | April 16, 2026 | Author: Oscar Castillo

1. Overview

A client-side web application to upload and process work-order hours earned data along with bundle definitions, presenting detailed tables and visual charts. Supports grouping work orders into bundles with metadata, sorting, and generating printable/exportable reports with charts.

2. Objectives

  • Enable users to upload two required CSV files (hours earned and bundle definitions) plus an optional EV Details CSV
  • Aggregate earned hours per work order by month, summing across operations
  • Support definition of bundles grouping multiple work orders, with associated metadata
  • Display tabular data grouped by bundle and sorted by part number and order number
  • Visualize data with charts: one series per root part number plus bundle total
  • Provide PDF/print export of tables and charts
  • Provide self-contained HTML snapshot export
  • Hide months prior to the bundle's first earned hours to improve readability
  • Identify unbundled work orders to help users maintain bundle definitions

3. Architecture Constraints

  • Entirely client-side; all processing and chart rendering happens in the browser
  • Single self-contained HTML file with embedded libraries
  • No external server calls; libraries loaded from embedded source (not CDN)
  • No persistent storage (cookies, localStorage, etc.)
  • Must work in modern browsers (Chrome, Edge)
  • Handle large datasets efficiently (36,000+ rows) without crashing or freezing
  • Clean, professional UI styling suitable for aerospace/engineering environments (navy and gray color palette, clear fonts, right-aligned numeric columns)

6. Functional Requirements

6.1 Data Processing

F-01 Sum hours earned per work order for each Year-Month, aggregating multiple operations in a month.

F-02 Map each work order to a bundle, including Program, Part Number, Part Description, and Serial Number metadata.

F-03 Validate that all work orders listed in the bundle definition exist in the hours data; if missing, display a warning.

F-04 Detect work orders appearing in multiple bundles and display an error (blocks rendering).

F-05 Identify unbundled work orders (in hours data but not in any bundle).

F-06 Support month names ("January", "February", etc.) and numeric months (1, 2, etc.) in the Month column via parseMonth().

6.2 Filters

F-10 "Data starts from" filter: Month/Year dropdowns. Default: January 2025. All data before the selected month is excluded from all tables and charts.

F-11 "Hide last month" toggle: Checkbox, checked by default. Removes the last month in the dataset from all views (incomplete month data).

F-12 Program filter chips: Clickable chips for each program found in bundle + EV Details data. Blue = included, grey strikethrough = excluded. Deselecting a program removes its work orders everywhere (bundled, unbundled, summary, CSV export).

F-13 Filters take effect when "Apply" is clicked (for start date/year) or immediately on toggle (hide last month, program chips).

6.3 Summary Section

F-20 Summary section at top of output, titled "Total Earned Hours Per Month".

F-21 Summary table: one row per bundle showing monthly totals, an "Unbundled (N WOs)" row, and a "Grand Total" row.

F-22 Each cell displays the hours value followed by a percentage label showing that value as a fraction of the Grand Total for that month.

F-23 Summary chart: one colored line per bundle, a purple dashed "Unbundled" line (if unbundled hours exist), and a navy dashed "Grand Total" line.

F-24 Chart title: "Total Earned Hours Per Month".

F-25 Bundle names in the summary table "Category" column are hyperlinks that scroll to the corresponding bundle section.

6.4 Region Averaging

F-30 Up to 3 averaging regions per chart (summary and each bundle chart independently).

F-31 Each region has Start and End month dropdowns.

F-32 On the chart canvas: shaded box with dashed border, dashed horizontal average line, and "Avg: X.X hrs/mo" label with white background.

F-33 Region colors: blue (#2563eb), amber (#b45309), green (#15803d).

F-34 Region controls hidden in snapshot and print; shaded boxes and labels remain on chart canvas.

F-35 Region average is computed from the "total" line of that chart (Grand Total for summary, Bundle Total for per-bundle).

F-36 When legend items are toggled off, the total line and region averages recalculate to exclude hidden datasets. A red "Excluding: ..." note appears at the top-right of the chart area.

F-37 Predefined regions are preserved in snapshots and restored with their start/end indices on snapshot load.

6.5 Per-Bundle Sections

F-40 One section per bundle, with heading showing bundle name (static, not clickable). Each section has an id="bundle-section-N" anchor for navigation from the summary table.

F-41 Chart displayed above data table. Chart always visible; data table collapsed by default.

F-42 A "Data Table" toggle bar (grey, below region controls) expands/collapses the data table with a triangle icon. Bundle heading is static.

F-48 "Back to Top" link in the region controls bar (right-aligned) smooth-scrolls to the top of the page.

F-49 "Back to Top" link persists when regions are added/removed (rendered by renderChartRegionControls for bundle charts).

F-43 Bundle chart groups work orders by root part number. Root = everything before last dash (e.g., 123456-12 and 123456-13 both become 123456-x).

F-44 Chart legend shows part description (first found for that root group) instead of root part code.

F-45 Bundle chart includes a dashed "Bundle Total" line.

F-46 Data table shows individual work orders: Program, Part Number (with description), Serial No, Order No, monthly hours, Total.

F-47 Work orders sorted within each bundle by Part Number (alphanumeric) then Order No.

6.6 Unbundled Work Orders

F-50 Collapsible section at bottom with amber header. Default: collapsed.

F-51 Only shows work orders that have earned hours within the allowed month range (respects start filter + hide-last-month).

F-52 If EV Details uploaded: groups by Program > Network > Part No + Plan Title hierarchy. Otherwise: flat alphabetical list.

F-53 WOs not found in EV Details shown under "(No EV Details Match)" subheading.

F-88 During drag, a semi-transparent blue selection rectangle with dashed border is drawn on the chart canvas.

6.9 Export Bundle Velocity (v1.2)

Revised in v1.2: previously per-WO averages. Capacity throughput is better characterized at the bundle (work-package) level because month-to-month WO earning is noisy (kitting, long-lead parts, cert availability, capacity). Bundle-level velocity aligns with how project managers track delivery commitments.

F-100 "Export Bundle Velocity" button in the action bar, visible after analysis runs.

F-101 Clicking the button reveals an inline settings panel with: lookback months dropdown (3, 6, 9, 12; default 6), exclude current month checkbox (default checked), Download CSV button, Cancel button, and period info label.

F-102 Export includes only bundles defined in the Bundle Definition CSV. Unbundled work orders are excluded (the separate "Export Unbundled CSV" remains for reference).

F-103 Bundle velocity (lookback) = mean of per-month bundle totals across the lookback window. Monthly total = sum of member WO hours for that month. Months with 0 hours count toward the mean (they represent months with no progress).

F-104 Std deviation = population std dev over per-month bundle totals. CV = stdDev / velocity; blank when velocity = 0.

F-104a Bundle velocity (last month) = the bundle's total hours in the single most recent month of the selected window. Provides a "current pace" goalpost independent of the lookback trend. StdDev and CV for the last-month figure are emitted blank (single-sample statistic).

F-105 Two-section CSV:
  Section 1 (one row per bundle): Bundle Name, Program, Lookback Months, Exclude Current Month, Period Start, Period End, Velocity Lookback (hrs/mo), StdDev Lookback, CV Lookback, Month Count Lookback, Last Month, Velocity Last Month (hrs/mo), StdDev Last Month, CV Last Month.
  Section 2 (one row per member WO): Bundle Name, Order No, Part Number, Part Description, Serial Number.
  Sections are separated by a blank line and each is prefixed with a # comment header.

F-106 Filename: bundle-velocity-YYYY-MM-DD.csv.

F-107 Member WO metadata sourced from the Bundle Definition CSV. Program is inferred per-bundle from the first member WO that carries a Program value.

F-108 Lookback and exclude-current-month controls dynamically update the period info label showing the actual month range and count.

9. Known Issues

RESOLVED (v1.1): "Hide last month" toggle was not working. Root cause: the orphan WO month-range extension loop in renderAll() called getMonthRange() without re-applying trimLastMonth(), re-adding the last month after it had been trimmed. Fixed by wrapping the expansion call with trimLastMonth().

No open bugs at this time.
